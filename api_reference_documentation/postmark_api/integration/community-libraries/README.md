# Postmark Community Libraries

## Overview

The Postmark developer community has created numerous third-party integrations across multiple programming languages and frameworks. These libraries are **not officially maintained by Postmark** and are contributed by community developers.

> "Every library below was contributed by a member of our API developer community. Please note that these libraries are not developed, maintained, or supported by Postmark directly."

For issues or feature requests, contact the respective project maintainers directly.

---

## Language-Specific Libraries

### Classic ASP

- **Simple Classic ASP Class for Postmark** — James Healey's implementation for integrating with the Postmark service

### Clojure

- **Clojure Binding for Postmark API** — Created by Steve Losh to enable functional programming language support

### ColdFusion

- **Postmark API Wrapper** — Matt Gifford's ColdFusion wrapper for API interaction
- **ColdFusion Component** — Jonathan Lane's alternative implementation for the CFML ecosystem

### Django

- **Anymail** — Mike Edmunds' unified email backend supporting "Mailgun, Mailjet, Postmark, SendGrid, SparkPost and more"

### Drupal

- **Drupal Module** — Luke Simmons and Deeson Online's integration for the Drupal CMS

### Elixir

- **Swoosh** — Email library for Elixir applications

### Erlang
